# Vercel deployment notes

The web app at `apps/web` is what deploys here. The Fastify API
at `apps/api` does NOT deploy here — host it on Railway or Fly.io
because it needs:

- A long-running container (Vercel functions cap at 60s)
- yt-dlp + ffmpeg binaries (Vercel doesn't allow custom binaries)
- Streaming responses larger than 4.5 MB (Vercel hard limit)
- Sustained outbound bandwidth (every download proxies through here)

## Vercel project settings

When importing the repo:

- Root directory: `apps/web`
- Framework preset: Next.js (auto-detected)
- Build command: `pnpm --filter @save47/web build` (set in vercel.json)
- Install command: `pnpm install --frozen-lockfile=false` (set in vercel.json)
- Output directory: `.next`

## Environment variables (Vercel → Project → Settings → Environment Variables)

```
NEXT_PUBLIC_API_URL=https://<your-railway-app>.up.railway.app
NEXT_PUBLIC_SITE_URL=https://save47.com
NEXT_PUBLIC_CF_TURNSTILE_SITE_KEY=0x4AAA...
NEXT_PUBLIC_APK_URL=https://github.com/<you>/save47/releases/latest/download/save47-latest.apk
NEXT_PUBLIC_APK_VERSION=1.0.0
```

Set these for both Production and Preview.
